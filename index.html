暂存区

  • 能使你“变更的”和“需要提交的”分开

  • 甚至支持一个文件只提交一部分

文件状态

暂存区的操作

  • git add <file>

    • 新增的文件(add the file)
    • 修改的文件(stage the file)

  • git reset HEAD <file>

    将变更从暂存区中移除(remove the file)

    HEAD为版本库“最年轻”的版本

  • git commit

    将暂存区内容提交

查看日志

  • git log

  • git log --stat

    查看变更文件及行数

版本号

  • 指纹字串——SHA-1哈希值

    长度为40的十六进制字符串

  • 用来标识存储的对象

  • 直接使用前5位即可

git的对象模型

  • Commit

    代表一次提交

  • Tree

    代表一个目录

  • Blob

    代表一个文件

  • tag

    标记一个提交

Commit

  • commit指向一个tree

    git show -s --pretty=raw <commitSHA1>

  • parent指向上一个commit

Tree

  • 指向一组tree或blob

    git ls-tree <treeSHA1>

Blob

  • 由文件内容计算出SHA1值

  • git show <blobSHA1>

    直接查看文件内容

  • 不包含文件名

  • git diff

    查看工作目录与HEAD的差异

  • git diff --staged

    查看暂存区与HEAD的差异

    Git 1.6.1以前使用: git diff --cached

启用色彩

  • git config --global color.ui true

    打开全部配色

撤销变更

  • git checkout -- <file>

    将工作目录的文件还原为HEAD

  • git reset HEAD^

    将HEAD版本撤销，HEAD^为当前最新版本的上一个版本

方便的commit

  • git commit -m <msg>

    直接添加注释信息并提交

  • 注释技巧

    • 第一行少量概要文字——标题
    • 第二行空行
    • 第三行开始详细描述——正文

  • git commit -a

    自动将所有变更和删除的文件添加到索引并提交，但不包括新增文件

临时仓库

  • 正在开发某功能，忽然一个紧急BUG需要修复

  • git stash

    把工作目录的变更压入栈中，并且工作目录还原到HEAD

  • git stash list

    查看栈中有哪些存储

  • git stash pop

    弹出最新压入的变更

查找BUG的利器

  • git blame <file>

  • git bisect
